# 🧪 Week 2 Assignment: Exploratory and Performance Testing

## 🎯 Learning Objectives
- 🔍 Apply exploratory testing techniques to uncover hidden bugs  
- ⚙️ Understand basic principles of performance testing  
- 📝 Practice documenting exploratory test sessions  
- 🏷️ Use GitHub Labels and Project Boards to track test outcomes  

---

## 📋 What You’ll Need
- 💻 A browser-based web app (provided in this repo)
- 🧾 Access to GitHub Issues and Project Board

---

## 📝 Submission Instructions
1. 🔎 Perform an exploratory testing session for **15–20 minutes**
2. 🐞 Submit **3 GitHub Issues** found during your session  
   - Use appropriate labels: `exploratory`, `ui`, `performance`, etc.
3. 🗂️ Create a `Test_Session_Report.md` file and include:
   - ✅ Test goals and areas explored  
   - 🐛 Summary of each bug discovered (with issue links)  
   - 📈 Notes on performance concerns (e.g., lag, load delays)
